# 模块：市场研究与洞察（Market Research）

## 1) market-landscape-scan（行业全景扫描）
- Use When：新业务立项、进入新赛道、季度战略更新
- Inputs（必填）：行业名称、目标市场、时间窗口、分析维度
- Inputs（可选）：预算区间、商业模式、重点渠道
- Outputs：行业规模与增速、关键趋势、主要玩家分层、机会与风险清单
- Workflow：定义范围 → 收集数据源 → 结构化归纳 → 机会优先级排序
- 示例输入：`行业=AI 外呼，市场=中国，时间=近 24 个月，维度=规模/竞争/渠道`

## 2) competitor-gap-analysis（竞品差距分析）
- Use When：竞品增长快、我方转化不及预期、需要制定对标与超越路径
- Inputs（必填）：我方产品简介、竞品列表、对比维度
- Inputs（可选）：目标客群、价格带、历史投放数据
- Outputs：竞品对比矩阵、能力差距排序、可突破点、90 天动作建议
- Workflow：建立评分框架 → 多维打分 → 差距诊断 → 输出改进路线
- 示例输入：`竞品=3-5 家，维度=价格/功能/内容/渠道/服务`

## 3) user-voice-mining（用户声音挖掘）
- Use When：需要从用户评论和客服记录中提炼真实需求与抱怨点
- Inputs（必填）：评论文本、平台来源、目标用户类型
- Inputs（可选）：时间范围、情绪标签、产品版本信息
- Outputs：高频痛点主题、情绪分布、用户原话证据、内容与产品建议
- Workflow：文本清洗 → 主题聚类 → 情绪分析 → 洞察映射到营销动作
- 示例输入：`数据源=小红书评论+客服工单，样本量=2000 条`
