# winback-automation-planner（召回自动化流程）

## 类型
Markdown Skill

## 说明
用于为沉睡用户设计多触点召回流程、消息模板和失败分支策略。

## 适用场景
- 沉睡用户增长、复购率下降、召回流程需要标准化

## 输入
### 必填
- 目标人群
- 可用渠道
- 权益资源
- 触达频控规则

### 可选
- 历史召回成功样本
- 渠道成本
- 客服承接能力

## 输出
- 召回流程图
- 消息模板
- 分支策略
- 监控指标

## 执行流程
1. 召回分群。
2. 设计触达编排。
3. 输出消息模板与失败分支。
4. 设定监控指标与止损规则。

## 示例
### 输入示例
`渠道=短信+企微+公众号，频控=7 天不超过 3 次`

### 输出示例
- 主流程：首触达、补触达、权益提醒
- 分支：已回流、未回流、投诉风险
- 指标：打开率、回流率、复购率
